# Railway 部署检查清单

## ✅ 部署前检查

- [x] `package.json` 存在且包含 `start` 脚本
- [x] `server.js` 使用 `process.env.PORT`（已配置）
- [x] `Procfile` 已创建（可选，但推荐）
- [x] `.gitignore` 已配置（排除 node_modules 和 data.json）
- [x] 所有必要文件都在项目中

## 📋 部署步骤

### 1. GitHub 准备（5分钟）
- [ ] 在 GitHub 创建新仓库
- [ ] 将代码推送到 GitHub
  ```bash
  git init
  git add .
  git commit -m "Initial commit"
  git remote add origin https://github.com/用户名/仓库名.git
  git push -u origin main
  ```

### 2. Railway 部署（3分钟）
- [ ] 访问 https://railway.app/ 并登录
- [ ] 点击 "New Project"
- [ ] 选择 "Deploy from GitHub repo"
- [ ] 选择你的仓库
- [ ] 等待部署完成

### 3. 获取访问地址（1分钟）
- [ ] 在 Railway Dashboard 找到生成的域名
- [ ] 复制 URL（格式：`https://xxx.up.railway.app`）
- [ ] 在浏览器中打开测试

### 4. 测试（2分钟）
- [ ] 访问网站首页
- [ ] 使用 `admin` / `123123` 登录
- [ ] 测试创建用户功能
- [ ] 测试创建奖项功能
- [ ] 测试概率设置功能

## 🎉 完成！

你的网站现在可以在公网访问了！
